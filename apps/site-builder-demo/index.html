Composed site: files + dynamic API + dynamic import

A SiteBuilder mounts two MemFilesApi instances — one for the client (HTML/JS/CSS), one for the server — plus a dynamic /api/* endpoint. The endpoint handler fetches server/api/index.js through the same service via callHttpService, wraps the response in a blob: URL, and import()s it to invoke its default export per request. Client fetches go through the relay ServiceWorker exactly like a normal HTTP site.
