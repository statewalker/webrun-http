Composed site: files + dynamic API + dynamic import

A SiteBuilder mounts two MemFilesApi instances — one for the client (HTML/JS/CSS), one for the server — plus a dynamic /api/* endpoint. The endpoint handler does await import(`${base}/server/api/index.js`): the imported module is served as JS by the same site handler (through the second files mount), evaluated by the browser, and its default export is invoked per request. Client fetches go through the relay ServiceWorker exactly like a normal HTTP site.
